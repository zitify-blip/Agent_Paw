DBA는 컬럼 자료형을 선정할 때 데이터의 길이 가변성·정밀도·저장 공간·인덱스 효율을 종합 고려한다.

---

## 1. 문자열 타입 선택

Database는 가변 길이는 VARCHAR, 고정 길이는 CHAR을 사용한다. 길이가 일정하지 않은 컬럼에 CHAR을 사용하는 것을 금지한다.

| 용도 | 권장 타입 | 사유 |
|---|---|---|
| 코드값(C02101 등) | CHAR(6) ~ CHAR(10) | 길이 고정 → 인덱스 스캔 효율 |
| 플래그('Y'/'N') | CHAR(1) | 1바이트 고정 |
| 사용자명·상품명 | VARCHAR(100) | 가변, 공간 절약 |
| 설명·메모 | VARCHAR(500) ~ VARCHAR(1000) | 가변 |
| 장문 텍스트(HTML 본문 등) | TEXT / MEDIUMTEXT | 65,535자 초과 대비 |
| 바이너리 | BLOB / LONGBLOB | 가급적 파일 서버 저장 후 URL만 DB 보관 권장 |

단, VARCHAR 길이는 실제 최대값의 1.5배 여유를 두어 산정한다. 과도한 VARCHAR(4000)은 인덱스 길이 제한에 저촉되므로 회피한다.

---

## 2. 정수 타입 선택

Database는 저장 범위와 테이블 예상 행 수를 기준으로 정수 타입을 선정한다.

| 타입 | 범위(Signed) | 저장 | 권장 용도 |
|---|---|---|---|
| TINYINT | -128 ~ 127 | 1B | 상태 코드(숫자형), enum 대체 |
| SMALLINT | -32,768 ~ 32,767 | 2B | 연도, 소규모 카운터 |
| INT | ±21억 | 4B | 일반 카운터·FK(소규모 테이블) |
| BIGINT | ±9.2 × 10^18 | 8B | PK(자동채번), 대용량 트랜잭션 FK |

DBA는 PK는 기본적으로 BIGINT를 사용하고, 명시 표기는 `BIGINT(20)`으로 기재한다. INT PK는 20억 행 근접 테이블에서 오버플로 위험이 존재한다.

---

## 3. 소수점·금액 타입 선택

금융·금액·수량 컬럼은 반드시 DECIMAL을 사용한다. FLOAT/DOUBLE은 이진 근사값으로 인해 정확한 집계·회계 용도에 부적합하다.

| 용도 | 타입 | 예시 |
|---|---|---|
| 금액(원화) | DECIMAL(15,2) | total_amt |
| 수량 | DECIMAL(10,3) | stock_qty |
| 비율(%) | DECIMAL(5,2) | tax_rt |
| 좌표·측정값 | DOUBLE | latitude, longitude |
| 근사 계산 과학 수치 | FLOAT | sensor_value |

단, 비율 컬럼은 `0.00 ~ 100.00`이면 DECIMAL(5,2), `0.0000 ~ 1.0000`이면 DECIMAL(5,4)로 구분한다.

---

## 4. 날짜·시간 타입 선택

Server는 시각 포함 여부에 따라 DATE와 TIMESTAMP를 구분한다.

| 타입 | 저장 | 범위 | 권장 용도 |
|---|---|---|---|
| DATE | 3B | 1000-01-01 ~ 9999-12-31 | 생일, 기준일자, 만료일 |
| DATETIME | 8B | 1000-01-01 00:00:00 ~ 9999-12-31 | 레거시 호환, 타임존 무관 업무 |
| TIMESTAMP | 4B (MySQL) / 8B (PostgreSQL) | 1970-01-01 ~ 2038-01-19 (MySQL) | 감사 로그(created_at/updated_at) |
| TIME | 3B | 00:00:00 ~ 23:59:59 | 영업 시간, 커트오프 시각 |

. 감사 컬럼(`created_at`·`updated_at`) : TIMESTAMP 사용 (UTC 저장, 표시 시 UTC+9 변환)
. 업무 일자(`start_dt`·`expire_dt`) : DATE 사용

단, MySQL TIMESTAMP는 2038년 오버플로 이슈가 존재하므로 신규 시스템은 DATETIME(6) 또는 BIGINT(UNIX epoch ms) 저장을 검토한다.

## 5. 기타 타입

| 타입 | 권장 용도 | 비고 |
|---|---|---|
| JSON | 반정형 속성(가변 스키마) | 인덱스·쿼리 제약 존재, 5개 이하 키 권장 |
| ENUM | 고정 소수 상태값 | 변경 시 ALTER 필요 → 코드 테이블 권장 |
| UUID(CHAR(36)) | 분산 시스템 PK | 인덱스 비용 높음, 필요 시만 사용 |
