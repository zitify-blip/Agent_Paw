DBA는 컬럼명을 데이터 의미와 타입을 즉시 추론 가능하도록 부여한다. 축약은 원칙적으로 금지하며, 접미어로 자료형 의미를 전달한다.

---

## 1. 기본 형식

| 기준 | 규칙 |
|---|---|
| 문자 집합 | 소문자 영문, 숫자, 언더스코어(_) |
| 단어 구분 | 스네이크 케이스(snake_case) |
| 최대 길이 | 30자 이하 |
| 복수형 | 금지. 컬렉션 값이라도 단수형 + 자료형 접미어로 표현한다 |

---

## 2. 의미 접미어 체계

System은 접미어를 통해 컬럼의 자료형·역할을 식별한다.

| 접미어 | 의미 | 예시 | 자료형 가이드 |
|---|---|---|---|
| _id | 식별자(PK/FK) | user_id, order_id | BIGINT / CHAR(36) |
| _cd | 코드값(코드 테이블 참조) | status_cd, category_cd | VARCHAR(10) |
| _nm | 명칭·이름 | user_nm, product_nm | VARCHAR(100) |
| _desc | 설명·상세 | order_desc | VARCHAR(500) |
| _dt | 날짜(시각 없음) | birth_dt, start_dt | DATE |
| _at | 일시(시각 포함, UTC 저장) | created_at, updated_at | TIMESTAMP |
| _yn | 예/아니오 플래그(레거시 호환) | use_yn, del_yn | CHAR(1) 'Y'/'N' |
| _cnt | 건수·개수 | order_cnt, item_cnt | INT |
| _amt | 금액 | total_amt, tax_amt | DECIMAL(15,2) |
| _qty | 수량 | stock_qty | DECIMAL(10,3) |
| _rt | 비율·율(%) | tax_rt, discount_rt | DECIMAL(5,2) |
| _url | URL | profile_url | VARCHAR(500) |
| _json | JSON 구조 저장 | payload_json | JSON / TEXT |

---

## 3. Boolean 컬럼 규칙

DBA는 Boolean 의미 컬럼을 접두어 `is_` / `has_` / `can_`로 부여한다. 레거시 호환 목적 외에는 `_yn` 사용을 지양한다.

| 접두어 | 의미 | 예시 |
|---|---|---|
| is_ | 상태 참/거짓 | is_active, is_deleted |
| has_ | 소유 여부 | has_child, has_profile |
| can_ | 권한·가능 여부 | can_edit, can_approve |

단, 금융·제조 업종 기존 시스템과 연동 시 `use_yn`, `del_yn` 형식을 허용한다. 혼용 시 테이블 단위로 일관성을 유지한다.

---

## 4. 날짜·일시 컬럼 구분

Server는 시각 정보 필요 여부에 따라 `_dt`(DATE)와 `_at`(TIMESTAMP)을 구분하여 사용한다.

| 구분 | 접미어 | 타입 | 예시 | 타임존 |
|---|---|---|---|---|
| 시각 없음(날짜만) | _dt | DATE | birth_dt, expire_dt | - |
| 시각 포함(이벤트 발생 시점) | _at | TIMESTAMP | created_at, paid_at | UTC 저장, 표시 시 UTC+9 변환 |

단, 배치 기준 일자처럼 시각이 무의미한 경우 반드시 `_dt`를 사용한다.

---

## 5. 축약 금지 원칙

DBA는 의미 전달이 불명확한 축약을 금지한다. 단, 업계 표준 축약은 허용한다.

| 축약 금지(사용 금지) | 권장 |
|---|---|
| usr_nm | user_nm |
| prd_cd | product_cd |
| ord_amt | order_amt |
| cust_no | customer_no |

허용 축약 : id(identifier), no(number), cd(code), nm(name), dt(date), yn(yes/no), amt(amount), qty(quantity), rt(rate), cnt(count), tel(telephone), addr(address).

---

## 6. FK 컬럼 명명

Server는 외래키 컬럼을 `[참조테이블 도메인]_id` 형식으로 부여한다. 접두어 `trx_`·`mst_`는 컬럼명에 포함하지 않는다.

```
참조 테이블 : mst_user
FK 컬럼명    : user_id   (❌ mst_user_id)
```

단, 동일 테이블에 같은 참조가 2회 이상 발생하는 경우 역할 접두어를 붙인다. 예) `creator_user_id`, `approver_user_id`.
