Agent Paw UI의 반응형 전환 규칙을 정의한다. Designer는 모바일 우선(Mobile-first) 원칙을 기반으로 설계하며 터치 타겟 최소 44px 규격을 준수한다.

---

## 1. 설계 원칙

. 모바일 우선(Mobile-first) : 최소 해상도 375px을 기준 레이아웃으로 설계하고, 해상도 증가에 따라 요소를 확장한다.
. Progressive Enhancement : 작은 화면에서 확실히 동작하는 UI를 기반으로, 큰 화면에서 부가 기능을 추가한다.
. Content Parity : 모든 브레이크포인트에서 핵심 콘텐츠·기능이 동일하게 제공되어야 한다. Mobile에서 기능 축소 불가.

## 2. 브레이크포인트 전환 체계

| 브레이크포인트 | Min Width | 레이아웃 전략 |
|---|---|---|
| mobile | 0px | 1단 세로 Stack, SideNav → Bottom Tab |
| tablet | 600px | 2단 Grid, SideNav → Collapsed Drawer |
| desktop | 1024px | 멀티 단 Grid, SideNav 상시 노출 |
| wide | 1440px | max-width 1440px 중앙 정렬 |

세부 컬럼·거터 값은 `레이아웃그리드.md §2` 를 따른다.

## 3. 주요 컴포넌트 전환 규칙

| 컴포넌트 | Mobile | Tablet | Desktop |
|---|---|---|---|
| Navigation | 하단 Tab Bar (4개 고정) | Drawer(Hamburger) | Side Nav 240px 고정 |
| Data Table | Card List 변환 | 가로 스크롤 Table | 고정폭 Table |
| Dialog | Full-screen Bottom Sheet | 중앙 Modal (480px) | 중앙 Modal (640px) |
| Form | 1열 Stack | 1~2열 혼용 | 2열 Grid |
| Action Button | Bottom Fixed Bar | Inline Button | Inline Button |
| Tab | Scroll 가능한 Chip Tab | 고정 Tab | 고정 Tab |

단, Data Table의 핵심 컬럼(ID·Name·Status)은 Mobile Card 변환 시에도 반드시 노출한다.

## 4. 터치 타겟 최소 규격

Apple HIG·Material Design 기준을 통합하여 아래 최소값을 적용한다.

| 요소 | 최소 크기 | 최소 간격 |
|---|---|---|
| 단독 Button | 44x44px | 8px (간격) |
| 아이콘 Button | 44x44px (Hit Area) | 8px |
| Checkbox·Radio | 20x20px (시각) / 44x44px (Hit Area) | 12px |
| 링크(인라인) | 높이 24px 이상 | 8px |
| Bottom Tab | 56px 높이 | - |

. 시각적 크기와 Hit Area가 다를 경우 Hit Area를 확장한다. (예: CSS `padding` 또는 `::before` 확장)
. Touch Target 중첩 금지 : 인접한 두 타겟의 Hit Area가 겹치지 않도록 8px 이상 간격을 확보한다.
※ Desktop 마우스 환경에서도 44px 최소 규격을 유지하여 일관성을 확보한다.

## 5. 가로·세로 전환

. 회전(Orientation) 변경 시 입력 상태·스크롤 위치·선택 값을 유지한다.
. Mobile Landscape(가로)는 독립 브레이크포인트로 간주하지 않는다. 세로 기준 설계를 Width만 확장하여 재사용한다.
. 비디오·지도 등 몰입형 콘텐츠만 Landscape 전용 레이아웃을 허용한다.

## 6. 이미지·미디어 반응형

| 유형 | 전략 |
|---|---|
| 제품 이미지 | `srcset`으로 1x·2x·3x 제공, `<picture>`로 WebP·AVIF 선호 |
| 아이콘 | SVG 단일 자산 사용 — 벡터로 자동 대응 |
| 배경 이미지 | `object-fit: cover` 기본, 포커스 포인트(`object-position`) 지정 |
| 비디오 | `aspect-ratio` 고정 — 16:9(가로) / 9:16(세로 Shorts) |

단, 대역폭 절약을 위해 Mobile 브레이크포인트에서는 2x까지만 로드한다. 3x는 Tablet 이상에서만 제공한다.

## 7. 반응형 검증 체크리스트

Designer·QA는 배포 전 아래 해상도에서 UI를 검증한다.

. 375x667 (iPhone SE) — Mobile 최저 대응 해상도
. 414x896 (iPhone 11/Pro Max) — Mobile 일반
. 768x1024 (iPad) — Tablet 세로
. 1024x768 — Tablet 가로
. 1440x900 — Desktop 기본
. 1920x1080 — Wide
