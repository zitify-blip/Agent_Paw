Agent Paw UI의 인터랙션 7상태 표현 규칙을 정의한다. Designer는 모든 인터랙션 컴포넌트(Button·Input·Card 등)에 대해 본 문서의 상태 체계를 일관되게 적용한다.

---

## 1. 7상태 정의

| 상태 | 코드 | 정의 | 트리거 |
|---|---|---|---|
| default | S01 | 기본 휴지 상태 | 사용자 조작 없음 |
| hover | S02 | 포인터가 컴포넌트 위에 위치 | `:hover` 발생 |
| active | S03 | 사용자가 누르고 있는 순간 | `:active` 발생 |
| focus | S04 | 키보드 포커스 획득 | `:focus-visible` 발생 |
| disabled | S05 | 사용 불가 상태 | `disabled` 속성 부여 |
| loading | S06 | 비동기 처리 진행 중 | API 호출 진행 |
| error | S07 | 유효성 실패·오류 | Validation 실패 |

단, Touch 디바이스는 hover 상태 표현을 생략하고 active 상태로 대체한다.

## 2. Button 상태별 스타일

Primary Button(Default size md) 기준 상태별 스타일 적용 예시이다.

| 상태 | 배경 | 텍스트 | Border | 추가 효과 |
|---|---|---|---|---|
| default | primary/500 #2E75B6 | neutral/0 #FFFFFF | none | - |
| hover | primary/700 #1F4E79 | neutral/0 #FFFFFF | none | shadow/sm |
| active | primary/900 #0F2E4A | neutral/0 #FFFFFF | none | transform: scale(0.98) |
| focus | primary/500 #2E75B6 | neutral/0 #FFFFFF | 2px primary/300 | outline-offset 2px |
| disabled | neutral/100 #EEF0F3 | neutral/500 #8A929B | none | cursor: not-allowed |
| loading | primary/500 #2E75B6 | transparent | none | Spinner 16px 중앙 |
| error | error/500 #C0392B | neutral/0 #FFFFFF | none | Shake 300ms |

## 4. Focus 상태 표현 원칙

. 모든 인터랙티브 요소는 focus 상태를 반드시 시각적으로 노출한다. `outline: none` 단독 사용 불가.
. Focus Ring 두께 : 2px 기본, 4px Halo(반투명) 권장
. Focus Ring 색상 : `primary/500` 또는 컴포넌트 Primary 색상
. Offset : 최소 2px 확보하여 Border와 겹치지 않는다.
. 상세 기준은 `접근성.md §3` 을 따른다.

## 5. Loading 상태 표현

| 컴포넌트 | 표현 방식 |
|---|---|
| Button | 텍스트 숨김 + 중앙 Spinner 16px |
| Input | 우측 Spinner 16px, 입력 차단 |
| Card·List | Skeleton Shimmer (neutral/100 → neutral/50 gradient, 1.5s loop) |
| Page 전체 | 중앙 Spinner 32px + "Loading..." 텍스트 |
| Data Grid | 상단 Progress Bar 2px + Row Skeleton |

단, 200ms 이내 응답이 예상되는 인터랙션은 loading 상태 노출을 생략한다.

## 6. Error 상태 표현

. 인라인 오류 : 필드 하단 Helper Text 영역에 error/500 색상으로 메시지를 노출한다.
. 토스트 오류 : System 전역 오류는 Toast 컴포넌트로 3초간 노출 후 자동 소멸한다.
. 치명적 오류 : Dialog 강제 노출 — User가 명시적으로 Dismiss할 때까지 유지한다.
. 오류 메시지 문구 : 원인과 해결 방법을 1문장 내로 제공한다. (예: `이메일 형식이 올바르지 않다. example@domain.com 형태로 입력한다.`)

## 7. 상태 전환 애니메이션

| 전환 | Duration | Easing |
|---|---|---|
| default ↔ hover | 150ms | ease-out |
| default ↔ active | 100ms | ease-in |
| default ↔ focus | 0ms (즉시) | - |
| default ↔ disabled | 200ms | ease-in-out |
| enter loading | 200ms | ease-in-out |

※ `prefers-reduced-motion: reduce` 설정 시 모든 상태 전환 애니메이션을 비활성화한다. 자세한 기준은 `접근성.md §5` 를 따른다.
