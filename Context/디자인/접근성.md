Agent Paw UI의 웹 접근성 기준을 정의한다. Designer·개발 팀은 WCAG 2.1 Level AA를 최소 준수 기준으로 삼으며 핵심 Flow에 대해 AAA를 지향한다.

---

## 1. 준수 기준

. 표준 : WCAG 2.1 (Web Content Accessibility Guidelines 2.1)
. 준수 수준 : Level AA (모든 화면 필수) / Level AAA (로그인·결제·신청 Flow 지향)
. 국내 기준 : 한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.2 병행 준수
. 검증 도구 : Axe DevTools, Lighthouse, WAVE, 스크린리더(NVDA·VoiceOver)

## 2. 컬러 대비

| 요소 | 최소 대비비 | 기준 |
|---|---|---|
| 본문 텍스트 (18pt 미만) | 4.5:1 | WCAG AA |
| 큰 텍스트 (18pt 이상·14pt Bold 이상) | 3:1 | WCAG AA |
| UI 컴포넌트·그래픽 | 3:1 | WCAG AA (비텍스트 콘텐츠) |
| Focus Indicator | 3:1 | WCAG AA |
| AAA 본문 | 7:1 | 지향 |

. 컬러만으로 정보를 전달 불가. (예: 필수 표시는 색상과 `*` 기호를 병기)
. Disabled 요소는 대비비 예외를 허용한다.
. 실제 값은 `컬러시스템.md §3` 의 Semantic 토큰을 참조한다.

## 3. Focus 표시

키보드 사용자가 현재 Focus 위치를 명확히 인지할 수 있도록 아래 기준을 따른다.

| 항목 | 기준 |
|---|---|
| Focus Ring 두께 | 최소 2px |
| Focus Ring 색상 | `primary/500` 또는 대비 3:1 이상 색상 |
| Focus Ring Offset | 최소 2px |
| 표시 방식 | `:focus-visible` 활용 — 마우스 클릭 시 숨김, 키보드 Tab 시 표시 |
| 금지 | `outline: none` 단독 선언 불가 |

세부 상태별 스타일은 `상태표현.md §4` 를 따른다.

## 4. 키보드 접근성

. 모든 인터랙티브 요소는 `Tab` 키로 접근 가능해야 한다.
. Tab Order는 시각적 순서와 DOM 순서가 일치해야 한다. `tabindex` 양수 값 사용 불가.
. Dialog·Dropdown 등 Overlay 컴포넌트는 포커스 트랩(Focus Trap)을 적용한다.
. `Esc` 키로 Overlay를 닫을 수 있어야 한다.
. Skip Link("본문 바로가기")를 페이지 최상단에 제공한다.

| 키 | 동작 |
|---|---|
| Tab / Shift+Tab | 다음/이전 요소로 이동 |
| Enter | Button·Link 활성화 |
| Space | Button 활성화, Checkbox 토글 |
| Arrow Up/Down | Select·Radio·Menu 내 이동 |
| Esc | Dialog·Dropdown·Tooltip 닫기 |

## 5. 모션·애니메이션

. `prefers-reduced-motion: reduce` 감지 시 모든 비필수 애니메이션을 즉시 비활성화한다.
. Flash·Blink 콘텐츠 : 초당 3회 이상 깜빡이는 콘텐츠 사용 불가.
. Parallax·자동 재생 : User가 일시정지할 수 있는 Control을 제공한다.

```css
/* ✅ 권장 */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}
```

## 6. 스크린리더 대응

. 모든 이미지는 의미에 따라 대체 텍스트(alt)를 제공한다. 장식 이미지는 `alt=""`로 명시하여 스크린리더가 건너뛰도록 한다.
. 아이콘 Button은 `aria-label` 필수 부여한다. (예: `aria-label="닫기"`)
. 입력 필드는 `<label>` 또는 `aria-labelledby`로 레이블을 연결한다. Placeholder 단독 사용 불가.
. 상태 변경(로딩 완료·에러 발생)은 `aria-live="polite"` 영역으로 안내한다.

## 7. ARIA 주요 패턴

| 패턴 | 필수 ARIA 속성 |
|---|---|
| Dialog | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` |
| Tab | `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` |
| Accordion | `aria-expanded`, `aria-controls` |
| Dropdown Menu | `role="menu"`, `role="menuitem"`, `aria-haspopup` |
| Form 오류 | `aria-invalid="true"`, `aria-describedby="error-id"` |
| Loading | `aria-busy="true"`, `role="status"` |
| Toast | `role="alert"` (긴급) / `role="status"` (일반) |

※ ARIA는 네이티브 HTML 시맨틱을 대체하지 않는다. `<button>` 사용이 가능하면 `<div role="button">` 대신 `<button>`을 사용한다.

## 8. 폼 접근성

. 필수 필드는 `aria-required="true"` 속성과 시각적 `*` 표기를 병행한다.
. 오류 메시지는 필드와 `aria-describedby`로 연결한다.
. 자동 완성 속성(`autocomplete`)을 적절히 제공한다. (예: `autocomplete="email"`)
. 오류 발생 시 해당 필드로 포커스를 이동한다.
