Figma 라이브러리의 컴포넌트·Variant·폴더 구조 명명 규칙을 정의한다. Designer와 개발 팀이 동일한 명칭 체계를 공유하여 핸드오프·코드 매핑 비용을 최소화한다.

---

## 1. 기본 명명 원칙

. 언어 : 영문 PascalCase를 기본으로 한다. 한글 컴포넌트명 사용 불가.
. 구분자 : 슬래시(`/`)로 계층을 구분한다. (예: `Button/Primary`)
. 약어 : 일반적으로 통용되는 약어만 허용한다. (예: `Nav`, `Btn` 금지 — `Navigation`, `Button` 사용)
. 동사 금지 : 컴포넌트명에 동사를 사용하지 않는다. (예: `Submit` 금지 — `ButtonSubmit` 가능)

## 2. 폴더 구조

Figma 라이브러리는 3계층을 유지한다. 4단계 이상 Nesting을 금지한다.

```
Library/
├── 01-Foundation/         ← Color·Typography·Spacing 등 토큰
├── 02-Elements/           ← Button·Input·Icon 등 원자 컴포넌트
├── 03-Components/         ← Card·Dialog·Navigation 등 조합 컴포넌트
├── 04-Patterns/           ← Header·Footer·SideNav 등 레이아웃 패턴
└── 05-Templates/          ← Page Template·Dashboard 등 완성 화면
```

| 계층 | 명명 형식 | 예시 |
|---|---|---|
| Category | `[Category]` | `Button`, `Input`, `Card` |
| Sub-category | `[Category]/[Kind]` | `Button/Primary`, `Input/Text` |
| Variant Set | `[Category]/[Kind]` (Variant Property로 분기) | `Button/Primary` + `size=md, state=hover` |

## 3. Variant Property 명명

Variant Property는 소문자 영문 kebab-case를 사용한다. Value는 kebab-case 소문자 단어로 통일한다.

| Property | 허용 Value | 사용 컴포넌트 |
|---|---|---|
| size | xs · sm · md · lg · xl | Button, Input, Avatar |
| state | default · hover · active · focus · disabled · loading · error | 모든 인터랙션 컴포넌트 |
| variant | primary · secondary · tertiary · ghost · danger | Button, Tag |
| type | outline · filled · duotone | Icon |
| density | compact · comfortable · spacious | Table, List |
| selected | true · false | Tab, Toggle, Chip |
| has-icon | leading · trailing · both · none | Button, Input |

단, 프로젝트 특수 Property가 필요한 경우 `컴포넌트네이밍.md`에 추가 기재 후 사용한다. 무단 추가 불가.

## 4. Variant 조합 순서

Variant Property는 아래 순서로 Figma에 등록한다. 순서 일관성이 유지되어야 Devs·QA의 필터 검색 효율이 확보된다.

. 1순위 : `variant` — 컴포넌트의 의미적 스타일
. 2순위 : `size` — 크기
. 3순위 : `state` — 상호작용 상태
. 4순위 : `has-icon` / `selected` 등 부가 속성

## 5. Layer 명명

Figma 내부 Layer는 의미 기반 명칭을 사용한다. 기본 Layer명(`Rectangle 1`, `Frame 23`)은 금지한다.

| Layer 유형 | 명명 형식 | 예시 |
|---|---|---|
| Container | `container-[용도]` | `container-button`, `container-card` |
| Slot | `slot-[용도]` | `slot-icon-leading`, `slot-label` |
| Text | `text-[역할]` | `text-label`, `text-helper` |
| Icon | `icon-[이름]` | `icon-chevron-down` |
| Background | `bg-[상태]` | `bg-default`, `bg-hover` |

## 6. Deprecated 컴포넌트 처리

. Figma 컴포넌트를 단종할 경우 즉시 삭제하지 않고 이름 앞에 `[Deprecated]`를 표기한다.
. 최소 1개 Minor 버전 이후 라이브러리에서 삭제한다.
. Deprecated 표기 시 Description에 대체 컴포넌트명을 반드시 명시한다.

```
❌  Button/Primary-Old         ← 이전 이름 그대로 유지
✅  [Deprecated] Button/Primary ← Description: "Use Button/Primary v2"
```
