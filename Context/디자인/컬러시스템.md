Agent Paw UI 전반에서 사용하는 컬러 토큰을 정의한다. Designer는 본 문서에 정의된 토큰만을 사용하며 임의 Hex 값을 직접 지정하는 것을 금지한다.

---

## 1. Primary 컬러

Primary는 브랜드 식별 및 주요 액션 버튼·링크에 사용한다.

| 토큰명 | Hex | 용도 |
|---|---|---|
| primary/50 | #E8F1FB | Background·Hover tint |
| primary/100 | #C5DCF5 | 비활성 Surface |
| primary/300 | #6BA8E0 | Secondary Action |
| primary/500 | #2E75B6 | 기본 Primary, 링크 |
| primary/700 | #1F4E79 | Pressed·Active 상태 |
| primary/900 | #0F2E4A | 다크 모드 강조 |

## 2. Secondary 컬러

보조 액션·강조 영역에 사용한다.

| 토큰명 | Hex | 용도 |
|---|---|---|
| secondary/300 | #F4B183 | Hover tint |
| secondary/500 | #ED7D31 | Secondary Button |
| secondary/700 | #C55A11 | Pressed 상태 |

## 3. Semantic 컬러

상태 전달을 위한 의미 기반 컬러를 정의한다. Designer는 Success·Warning·Error·Info 4종 외 색상을 상태 표현에 사용 불가하다.

| 토큰명 | Hex | 용도 | 대비(AA) |
|---|---|---|---|
| success/500 | #2E8B57 | 성공 메시지·완료 배지 | 4.8:1 |
| warning/500 | #D99E0B | 주의 Alert·경고 배지 | 4.6:1 |
| error/500 | #C0392B | 에러 메시지·유효성 실패 | 5.2:1 |
| info/500 | #2E75B6 | 정보 안내·Tooltip | 4.7:1 |

단, Semantic 컬러를 장식 목적으로 사용하는 것을 금지한다.

## 4. Neutral 컬러

텍스트·배경·보더에 사용하는 중립 팔레트를 정의한다.

| 토큰명 | Hex | 용도 |
|---|---|---|
| neutral/0 | #FFFFFF | 기본 배경 |
| neutral/50 | #F7F8FA | Subtle 배경 |
| neutral/100 | #EEF0F3 | Divider·Card 배경 |
| neutral/300 | #C9CED5 | Border·Disabled Surface |
| neutral/500 | #8A929B | Placeholder·Caption |
| neutral/700 | #3F464E | Body 본문 텍스트 |
| neutral/900 | #1A1D21 | Heading 텍스트 |

## 5. 라이트·다크 모드 토큰 매핑

System이 OS의 `prefers-color-scheme` 값을 감지하여 자동 전환한다.

| Semantic Token | Light Mode | Dark Mode |
|---|---|---|
| bg/base | neutral/0 (#FFFFFF) | neutral/900 (#1A1D21) |
| bg/surface | neutral/50 (#F7F8FA) | #262A30 |
| bg/elevated | neutral/0 (#FFFFFF) | #2E333A |
| text/primary | neutral/900 (#1A1D21) | #F0F2F5 |
| text/secondary | neutral/700 (#3F464E) | #B8BEC6 |
| text/muted | neutral/500 (#8A929B) | #8A929B |
| border/default | neutral/100 (#EEF0F3) | #363B42 |
| border/strong | neutral/300 (#C9CED5) | #4A5058 |
| action/primary | primary/500 (#2E75B6) | primary/300 (#6BA8E0) |

단, 로고·브랜드 일러스트 자산은 라이트·다크 모드에서 별도 버전을 사용한다.
※ 대비는 `접근성.md §2` 기준 AA 4.5:1 이상을 만족해야 한다.

## 6. 사용 제약

. Designer는 Figma Variables 컬렉션(`Color/Light`, `Color/Dark`) 내 토큰만 사용한다.
. 개발 전달 시 CSS Variable 명명은 `--color-[category]-[scale]` 형식을 따른다. (예: `--color-primary-500`)
. 투명도(Alpha)는 0·5·10·20·40·60·80 7단계만 허용한다.
