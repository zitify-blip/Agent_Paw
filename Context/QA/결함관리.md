결함(Defect) 발견부터 종료까지의 라이프사이클·상태 전이·재오픈 기준을 정의한다. 결함 관리 도구는 JIRA 를 표준으로 한다.

---

## 1. 결함 라이프사이클

### 1.1. 상태 정의

| 상태 | 코드 | 의미 | 진입 조건 |
|---|---|---|---|
| Open | D01 | 결함 신규 등록, 수정 착수 전 | QA가 결함을 JIRA에 등록한다 |
| In Progress | D02 | 개발자가 수정 착수 | 담당 개발자가 티켓을 본인에게 할당한다 |
| Resolved | D03 | 코드 수정·PR 머지 완료 | 개발자가 수정 커밋을 Release 브랜치에 머지한다 |
| Verified | D04 | QA 재현 테스트 통과 | QA가 수정 빌드에서 재현 실패를 확인한다 |
| Closed | D05 | 종료. 리그레션 통과 확인 완료 | QA가 해당 빌드의 리그레션을 통과시킨다 |
| Reopened | D06 | 재오픈. 수정 후 재현됨 | 추가 검증에서 동일 결함이 재현된다 |
| Rejected | D07 | 결함 아님. 기획 의도 또는 환경 이슈 | PM·개발자가 비결함으로 판정한다 |

### 1.2. 상태 전이

| 현재 상태 | 다음 상태 | 전이 주체 | 전이 조건 |
|---|---|---|---|
| Open | In Progress | 개발자 | 수정 착수 |
| Open | Rejected | 개발자 + PM | 기획 의도·환경 이슈로 확인 |
| In Progress | Resolved | 개발자 | 수정 PR 머지 |
| In Progress | Open | 개발자 | 수정 불가, 재할당 필요 |
| Resolved | Verified | QA | 재현 테스트 통과 |
| Resolved | Reopened | QA | 재현 테스트에서 동일 증상 확인 |
| Verified | Closed | QA | 리그레션 스위트 통과 |
| Verified | Reopened | QA / 사용자 | 프로덕션에서 재발견 |
| Closed | Reopened | QA / 사용자 | 동일 원인 결함 재발생 |
| Reopened | In Progress | 개발자 | 재수정 착수 |
| Rejected | Open | QA | 판정 이의 제기 |

## 2. 결함 등록 필수 정보

결함 등록 시 아래 필드를 반드시 기록한다. 미기재 시 System이 저장을 불가하게 처리한다.

| 필드 | 필수 | 내용 |
|---|---|---|
| 제목 | Y | [모듈명] 한 줄 증상 요약 |
| 심각도 | Y | Sev1 / Sev2 / Sev3 / Sev4 |
| 우선순위 | Y | P0 / P1 / P2 / P3 |
| 발견 환경 | Y | DEV / STG / QA / PRD |
| 발견 빌드 | Y | 빌드 번호 또는 커밋 해시 |
| 재현 단계 | Y | 번호 리스트. System이 결함을 재현하는 최소 단계 |
| 기대 결과 | Y | 정상 동작 서술 |
| 실제 결과 | Y | 관측된 동작 서술 |
| 첨부 | N | 스크린샷·로그·영상 |
| 연관 TC | N | 발견된 테스트 케이스 ID |

## 3. 재오픈 기준

### 3.1. 재오픈 조건

. 동일 재현 단계에서 동일 증상 재현 : 즉시 Reopened 전환한다.
. 수정 범위 외 사이드이펙트 발생 : 기존 티켓을 Reopened 하지 않고 신규 결함으로 등록한다.
. 프로덕션에서 Closed 결함 재발생 : 해당 티켓을 Reopened 하고 심각도를 1단계 상향한다.

### 3.2. 재오픈 제한

. Closed 후 30일 경과 결함은 재오픈 대신 신규 티켓으로 등록한다. 단, 참조 링크로 원본 티켓을 연결한다.
. Rejected 판정 결함의 이의 제기는 판정 후 7일 이내에만 가능하다. 경과 시 신규 등록으로 대체한다.

---

## 4. 결함 종료 기준

. Closed 조건 : 재현 실패 확인 + 관련 리그레션 스위트 통과 + RCA 기록(Sev1·Sev2 한정).
. System이 빌드 배포 완료 시 Verified → Closed 로 일괄 전환한다.
. Sev1 결함은 RCA 문서가 JIRA에 첨부되지 않으면 Closed 전환이 불가하다.

---

## 5. 메트릭 관리

QA는 주간 리포트에 아래 지표를 포함한다.

| 지표 | 산식 | 목표 |
|---|---|---|
| Defect Escape Rate | PRD 발견 결함 / 전체 결함 | 5% 이하 |
| 평균 수정 시간(MTTR) | Open → Resolved 소요 시간 평균 | Sev1 2시간, Sev2 24시간 이내 |
| 재오픈율 | Reopened 건수 / Closed 건수 | 10% 이하 |
| Rejected 비율 | Rejected / 전체 등록 | 15% 이하 |
