테스트 계정·샘플 데이터·민감정보 마스킹·데이터 초기화 절차를 정의한다. 테스트 데이터는 개인정보보호법(PIPA) 및 내부 정보보안 정책을 준수한다.

---

## 1. 테스트 데이터 유형

### 1.1. 유형별 구분

| 유형 | 정의 | 출처 | 보관 위치 |
|---|---|---|---|
| 시드 데이터 | 고정 기준 데이터(코드·권한·기본 설정) | 기획서 기반 수작업 | `test-data/seed/` |
| 합성 데이터 | Faker 기반 생성 가상 데이터 | 스크립트 생성 | `test-data/synthetic/` |
| 익명화 데이터 | PRD 실데이터 마스킹본 | PRD 이관 + 마스킹 | `test-data/masked/` |
| 전용 계정 | 역할별 고정 테스트 계정 | QA 관리 | 1Password 공유 볼트 |

### 1.2. 환경별 데이터 배치

| 환경 | 시드 | 합성 | 익명화 | 실데이터 |
|---|---|---|---|---|
| DEV | 사용 | 사용 | 불가 | 불가 |
| QA | 사용 | 사용 | 제한적 사용 | 불가 |
| STG | 사용 | - | 사용 | 불가 |
| PRD | 사용 | 불가 | 불가 | 사용 |

## 2. 테스트 계정 관리

### 2.1. 표준 테스트 계정

| 계정 ID | 역할 | 용도 | 비고 |
|---|---|---|---|
| test_admin_01 | 관리자 | 관리자 기능 전체 검증 | 권한 변경 금지 |
| test_operator_01 | 운영자 | 운영 기능 검증 | - |
| test_user_01 | 일반 사용자 | 핵심 사용자 여정 검증 | - |
| test_user_vip_01 | VIP 사용자 | VIP 전용 기능 검증 | 등급 자동 갱신 차단 |
| test_user_new_01 | 신규 사용자 | 온보딩 검증 | 테스트 종료 시 초기화 |

### 2.2. 계정 관리 규칙

. 계정 비밀번호 : System이 분기 1회 일괄 갱신한다. 갱신 이력은 1Password에 기록한다.
. 개인 계정 사용 금지 : 개발자·QA가 개인 실명 계정으로 테스트하는 것을 불가하다.
. 계정 공유 : 테스트 계정 정보는 사내 1Password 공유 볼트에서만 열람 가능하며, 외부 전파를 제한한다.

## 3. 민감정보 마스킹

### 3.1. 마스킹 대상·방식

| 필드 | 원본 예시 | 마스킹 방식 | 마스킹 결과 |
|---|---|---|---|
| 이름 | 홍길동 | 성만 유지 + 무작위 | 홍**, 김** |
| 전화번호 | 010-1234-5678 | 중간 자리 * 치환 | 010-****-5678 |
| 이메일 | user@domain.com | 로컬파트 해시 | a1b2c3@domain.com |
| 주민번호 | 900101-1234567 | 완전 파기 | (저장하지 않음) |
| 주소 | 상세 주소 전체 | 시·군·구 까지만 유지 | 서울시 강남구 |
| 카드번호 | 1234-5678-9012-3456 | 앞 6자리 + 뒤 4자리만 유지 | 123456-****-3456 |

### 3.2. 마스킹 절차

. PRD → STG 이관 시 : System이 마스킹 스크립트를 자동 실행한다.
. 마스킹 검증 : QA가 이관 직후 샘플 100건을 추출하여 마스킹 누락 여부를 검증한다.
. 마스킹 실패 처리 : 원본 노출 발견 시 즉시 STG 환경을 격리하고 데이터를 전량 파기한다.

단, 주민번호·비밀번호·OAuth 토큰 등은 마스킹이 아닌 완전 파기를 적용한다.

## 4. 데이터 초기화 절차

### 4.1. 초기화 주기

| 환경 | 주기 | 방식 | 소요 시간 |
|---|---|---|---|
| DEV | 요청 시 | 수동 실행 | 10분 |
| QA | 주 1회(월요일 05:00) | 자동 스크립트 | 30분 |
| STG | 스프린트 시작 시 | 자동 + 수동 검증 | 2시간 |

### 4.2. 초기화 단계

. 1단계 : System이 기존 테이블을 TRUNCATE 또는 논리 삭제한다.
. 2단계 : System이 시드 데이터 SQL을 INSERT한다.
. 3단계 : System이 합성 데이터 스크립트를 실행한다(QA 환경 한정).
. 4단계 : QA가 샘플 검증 후 초기화 완료를 Slack `#qa-env` 채널에 공지한다.

단, 진행 중인 테스트가 있으면 초기화를 연기하며, QA 리드 승인 후 강제 진행 가능하다.
