본 문서는 `.env` 파일 관리, 시크릿 분리, 환경별 분기(dev/stg/prd) 규칙을 정의한다. 환경변수는 코드와 설정을 분리하는 12-Factor App의 핵심이다.

---

## 1. 환경 구분

| 환경 | 약칭 | 용도 | 데이터 |
|---|---|---|---|
| Development | `dev` | 로컬·개발 서버 | Mock / Seed 데이터 |
| Staging | `stg` | QA·UAT | 운영 복제 또는 익명화 데이터 |
| Production | `prd` | 실 운영 | 실데이터 |

. 환경 식별자는 `NODE_ENV` 또는 `APP_ENV` 변수로 구분한다.
. 환경별 기본값은 코드에 하드코딩을 불가하다.

## 2. 파일 구조

```
project-root/
├── .env                  ← 실제 로컬 변수 (Git 제외)
├── .env.example          ← 변수 목록 템플릿 (Git 포함)
├── .env.development      ← 개발 환경 기본값 (Git 포함, 비밀 없음)
├── .env.staging          ← 스테이징 환경 기본값 (Git 포함, 비밀 없음)
├── .env.production       ← 운영 환경 기본값 (Git 포함, 비밀 없음)
└── .gitignore            ← .env 제외 선언
```

. `.env` 와 `.env.*.local` 은 반드시 `.gitignore` 에 포함한다.
. `.env.example` 은 모든 변수명을 포함하되 값은 `xxx` 또는 공란으로 둔다.

## 3. 변수 네이밍

. 형식 : `SCREAMING_SNAKE_CASE` 를 사용한다.
. 접두사 : 모듈·도메인별로 접두사를 부여한다. 예) `DB_`, `AUTH_`, `STRIPE_`.
. 금지 : 한국어·소문자·하이픈 사용을 금지한다.

```
❌  database-url / databaseUrl / 데이터베이스URL
✅  DB_URL, DB_HOST, DB_PORT, DB_NAME
```

## 4. 환경변수 분류

| 분류 | 예시 | Git 커밋 | 저장소 |
|---|---|---|---|
| 공개 설정 | `APP_PORT`, `LOG_LEVEL` | ✅ `.env.*` | Git |
| 비공개 설정 | `FEATURE_FLAGS` | ❌ | Secret Manager |
| 시크릿 | `DB_PASSWORD`, `JWT_SECRET`, `API_KEY` | ❌ 절대 금지 | Secret Manager (AWS Secrets Manager / Vault / GCP Secret Manager) |

. 시크릿은 반드시 Secret Manager에 저장한다. 파일·DB·코드에 평문 저장을 절대 금지한다.
. 시크릿 로테이션 주기 : 최소 90일.

## 5. 로딩·검증 규칙

. 애플리케이션 시작 시 필수 환경변수 존재 여부를 System이 검증한다.
. 누락 시 시작을 중단한다(fail-fast). 기본값으로 대체하지 않는다.
. 타입 검증(숫자·불리언·URL 형식)을 수행한다.

```typescript
// 환경변수 스키마 검증 예시
const env = z.object({
  APP_PORT: z.coerce.number().int().positive(),
  DB_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
}).parse(process.env);
```

## 6. 환경별 분기 원칙

. 환경 분기 로직은 진입점(entry point)에 1회만 배치한다.
. 비즈니스 로직 내부에서 `if (env === 'prd')` 분기를 금지한다.
. Feature Flag를 사용하여 환경별 기능 활성화를 제어한다.

```
❌  if (process.env.NODE_ENV === 'production') { sendEmail() }
✅  if (featureFlags.EMAIL_NOTIFICATION_ENABLED) { sendEmail() }
```

## 7. 금지 사항

. 시크릿을 `.env.example`·Git·Slack·이슈 본문에 기록하는 것을 절대 금지한다.
. 로그 출력에 환경변수 전체를 dump하는 것을 금지한다. 개별 필요 변수만 출력한다.
. 운영 시크릿을 개발자 로컬에 다운로드하는 것을 금지한다. 로컬은 Mock·테스트 계정을 사용한다.
. `.env` 파일을 동료에게 이메일·메신저로 전송하는 것을 절대 금지한다.

단, 신규 개발자 온보딩 시 `.env.local.template` 을 Secret Manager에서 복원하는 절차를 별도 안내한다.
