## 1. 공백 및 들여쓰기

. 들여쓰기 : 스페이스 4칸을 기본으로 사용한다. JS/TS/JSON/YAML은 스페이스 2칸을 사용한다.
. 탭 문자 사용을 금지한다. 에디터는 Tab → Space 치환을 활성화한다.
. 행 끝 공백 : 모든 행 끝 공백(trailing whitespace)을 제거한다.
. 파일 끝 : 파일은 반드시 개행(`\n`)으로 종료한다.

## 2. 한 줄 길이

| 언어군 | 최대 길이 | 비고 |
|---|---|---|
| Python | 120자 | PEP 8의 79자를 확장 적용한다 |
| JavaScript / TypeScript | 100자 | Prettier 기본값 80을 확장한다 |
| Java / C# | 120자 | |
| SQL | 120자 | JOIN·WHERE 절은 의미 단위로 줄바꿈한다 |

. 초과 시 System이 포매터 오류로 표시한다. 의미 단위로 개행한다.

## 3. 괄호 및 공백

. 함수 호출 : `foo(a, b)` — 함수명과 `(` 사이에 공백 불가하다.
. 제어문 : `if (cond)` — 키워드와 `(` 사이에 공백을 둔다.
. 콤마 뒤 : 반드시 공백 1칸을 둔다. 앞에는 공백을 두지 않는다.
. 연산자 : 이항 연산자(`+`, `-`, `=`, `==` 등) 양쪽에 공백 1칸을 둔다.
. 중괄호 : 여는 중괄호 `{` 는 같은 줄에 배치한다(K&R 스타일).

```
❌  if(x==1){ doSomething() ;}
✅  if (x == 1) { doSomething(); }
```

## 4. 따옴표

. 문자열 : 프로젝트 기본 따옴표를 통일한다. JS/TS는 `'` 작은따옴표, Python은 `"` 큰따옴표를 사용한다.
. 템플릿 리터럴은 문자열 보간·멀티라인 시에만 사용한다.

## 5. 주석

. 주석은 한국어 또는 영어 중 프로젝트 기준을 따른다. 본 프로젝트는 한국어 주석을 허용한다.
. TODO 주석 : `// TODO(user_id): 내용 (YYYY-MM-DD)` 형식으로 작성자·기한을 명시한다.
. FIXME 주석은 반드시 이슈 번호와 연결한다.

단, 자명한 코드에는 주석을 달지 않는다. 주석은 "왜(Why)" 를 설명하고 "무엇(What)" 은 코드로 표현한다.

## 6. import·require 순서

1. 표준 라이브러리
2. 외부(third-party) 라이브러리
3. 사내 공용 모듈
4. 프로젝트 내부 상대 경로

. 각 그룹 사이에 빈 줄 1줄을 둔다.
. 알파벳 순으로 정렬한다.

## 7. 금지 사항

. 하드코딩된 매직 넘버·문자열을 금지한다. 상수로 분리한다.
. 300줄을 초과하는 함수·메서드를 금지한다. 초과 시 분리한다.
. 3단계를 초과하는 중첩(depth)을 지양한다. Early return·Guard clause를 사용한다.
. 주석 처리된(dead) 코드의 커밋을 금지한다. Git 이력으로 추적한다.
