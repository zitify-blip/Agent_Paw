---

## 1. 표기법 선택

| 대상 | 표기법 | 예시 |
|---|---|---|
| 변수·함수 (JS/TS/Java/C#) | camelCase | `userName`, `getUserList()` |
| 변수·함수 (Python) | snake_case | `user_name`, `get_user_list()` |
| 클래스·타입·인터페이스 | PascalCase | `UserService`, `OrderRepository` |
| 상수 | SCREAMING_SNAKE_CASE | `MAX_RETRY_COUNT`, `API_BASE_URL` |
| 파일 (소스) | 언어 관례에 따름 | `userService.ts`, `user_service.py` |
| 파일 (컴포넌트) | PascalCase | `UserProfile.tsx`, `OrderCard.vue` |
| 디렉토리 | kebab-case 또는 snake_case | `user-service/`, `order_module/` |
| DB 테이블·컬럼 | SNAKE_CASE | `user_account`, `created_at` |
| 환경 변수 | SCREAMING_SNAKE_CASE | `DATABASE_URL`, `JWT_SECRET` |

## 2. 의미 규칙

### 2.1. 동사 접두사 (함수)

. 조회 : `get`, `find`, `fetch`, `load`
. 생성 : `create`, `make`, `build`, `generate`
. 갱신 : `update`, `set`, `modify`
. 삭제 : `delete`, `remove`, `clear`
. 판정 : `is`, `has`, `can`, `should` (boolean 반환)
. 변환 : `to`, `parse`, `convert`, `format`

```
❌  userProcess()
✅  createUser() / updateUserStatus() / isUserActive()
```

### 2.2. 불리언 변수

. `is`, `has`, `can`, `should` 접두사를 사용한다.
. 부정 의미(`isNotValid`)는 금지한다. 긍정 의미(`isInvalid`)로 작성한다.

### 2.3. 약어·축약

. 일반 통용 약어(`URL`, `ID`, `HTTP`, `API`)만 허용한다. 사내 독자 약어는 금지한다.
. 약어는 표기법에 따라 표기한다 : `userId` (camelCase), `UserID` (PascalCase 한정 허용).

## 3. 금지 네이밍

| 금지 | 사유 | 허용 |
|---|---|---|
| `data`, `info`, `temp`, `tmp` | 의미 불명확 | 구체 도메인명 사용 |
| `arr`, `list`, `obj` | 타입만 표현 | `userList`, `orderItems` |
| `a`, `b`, `c`, `x` | 문맥 부재 | 의미 있는 이름 사용 |
| `flag`, `check` | 용도 불명확 | `isEnabled`, `hasPermission` |
| `Manager`, `Helper`, `Util` 남용 | 책임 불분명 | 구체 역할명 사용 |

단, 반복문 인덱스(`i`, `j`, `k`)와 수학 공식 구현(`x`, `y`) 은 예외적으로 허용한다.

## 4. DB 네이밍

. 테이블명 : 단수형을 사용한다. `user` (O), `users` (X).
. 기본키 : `id` 를 사용한다. 외래키는 `[테이블]_id` 형식을 사용한다. 예) `user_id`.
. 생성·수정 시간 : `created_at`, `updated_at`, `deleted_at` 으로 통일한다.
. 불리언 컬럼 : `is_`, `has_` 접두사를 사용한다. 예) `is_active`.

## 5. 파일명

. 소스 파일명은 내부 주요 export 식별자와 일치시킨다.
. 테스트 파일 : `[원본].test.ts`, `test_[원본].py` 형식을 사용한다.
. 타입 정의 파일 : `[원본].types.ts` 또는 `[원본].d.ts` 를 사용한다.
