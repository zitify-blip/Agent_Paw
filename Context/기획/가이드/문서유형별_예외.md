## 1. 유형별 예외 허용 사항

| 문서 유형 | 적용 규칙 | 예외 허용 사항 |
|---|---|---|
| 기획서 | 전체 적용 | 없음 |
| 기술명세서 | 전체 적용 | 없음 |
| API명세서 | 전체 적용 | 없음 |
| 테스트케이스 | 전체 적용 | 없음 |
| 릴리즈노트 | 전체 적용 | 대외 배포 시 "~습니다" 체 허용 (별도 버전 작성) |
| 회의록 | 전체 적용 | 발언 인용 시 원문 유지 허용 |
| 주간보고서 | 전체 적용 | 없음 |
| 사용자스토리 | 전체 적용 | 페르소나 서술 시 감정 표현 허용 |
| 제안서 | 전체 적용 | 배경(Why) 섹션에 한해 감정·주관 표현 허용 (예: "~필요 증대") |
| 요구사항정의서 | 전체 적용 | "확인 필요 사항" 열은 의문형(~인가, ~필요?) 허용 |
| 운영정책 | 전체 적용 | 약관 연계 조문은 경어체(~합니다) 허용. 단, "회사"·"회원"·"약관" 등 큰따옴표 용어 정의 선행 필수 |
| 스토리보드 | 전체 적용 | UI Description 은 명사형 종결(~버튼, ~노출, ~활성화) 허용 |
| 기능정의서 | 전체 적용 | 기능 정의 셀 내부 bullet 은 `•` 또는 `-` 사용 허용 |
| 협업프로세스 | 전체 적용 | 프로세스 다이어그램 내 노드 라벨은 명사구 허용 |
