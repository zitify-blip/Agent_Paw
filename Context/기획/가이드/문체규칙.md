## 1. 종결 어미

. 사용 : ~한다, ~이다, ~된다, ~불가하다
. 금지 : ~합니다, ~할 수 있습니다, ~됩니다

### 1.1. 경어체 → 평서체 전환 매핑

과거 IGS 시기(2015~2019) 문서에서 빈번히 나타난 경어체 문장을 평서체로 전환할 때 아래 매핑을 사용한다.

| 경어체 (금지) | 평서체 (사용) |
|---|---|
| ~이 가능합니다 | ~이 가능하다 / ~할 수 있다 |
| ~해 주시기 바랍니다 | ~한다 / ~해야 한다 |
| ~확인하여 주시기 바랍니다 | ~확인한다 |
| ~드립니다 / 안내하여 드립니다 | ~안내한다 / ~발송한다 |
| ~이루어집니다 | ~진행된다 / ~처리된다 |
| ~되어 있습니다 | ~되어 있다 |
| ~이용하실 수 있습니다 | ~이용한다 / ~이용 가능하다 |
| ~주어야 합니다 / ~해 주셔야 합니다 | ~해야 한다 / ~선행되어야 한다 |
| 꼭 필요한 | 필수 (명사형) |

단, 외부 고객 공지문·릴리즈노트 대외 배포 버전은 `문서유형별_예외.md` 에 따라 경어체를 별도 버전으로 병행 허용한다.

## 2. 필수 어휘

| 금지 표현 | 허용 표현 |
|---|---|
| 보여준다, 보인다 | 노출한다, 출력한다 |
| 바꾼다 | 갱신한다, 설정한다, 해제한다, 전환한다 |
| 넣는다 | INSERT한다, 저장한다, 기록한다 |
| 지운다 | 삭제한다, 파기한다, 논리 삭제한다 |
| 안 된다, 못한다 | 불가하다, 제한한다, 선행되어야 한다 |
| 좋은, 적절한, 충분히 | 수치나 명확한 상태 값으로 표현한다 |

## 3. 주어 명시

. 모든 동작 서술 시 행위 주체를 문장 앞에 명시한다.
. 허용 주어 : Client, Server, Admin, User, System, Database, [역할명]

```
(X) 로그인 버튼을 클릭하면 인증이 진행된다.
(O) User가 로그인 버튼을 클릭하면, Client가 Server에 인증 요청을 전송한다.
```

## 4. 수치 및 코드 병기

. 상태값은 반드시 코드와 함께 표기한다.
. 수치는 단위까지 명시한다.

```
(X) 대기 상태 / 충분한 기간
(O) 대기(C02101) / 발급일로부터 1년
```

## 5. 예외 및 제약 조건

. 예외 조건은 해당 항목 하단에 별도 행으로 기술한다.
. `단,` 또는 `※` 으로 시작한다.

## 6. 데이터 지향 서술

. DBA/System Architect 관점을 유지한다.
. 데이터의 생성(Create), 갱신(Update), 삭제(Delete), 보존(Retain) 등 생명주기 관점에서 서술한다.
. 정상 흐름(Happy Path) 외에 실패, 취소, 롤백, 철회 시나리오를 반드시 포함한다.

## 7. 시각적 표현

. 상태 전이, 권한 매트릭스, 조건별 분기 로직은 줄글 대신 반드시 테이블로 표현한다.
