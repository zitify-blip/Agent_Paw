## 1. 표준 파일명 형식

```
[시스템명]_[문서유형]_v[Major].[Minor].[Patch].[확장자]
```

## 2. 구성 요소

### 2.1. 시스템명

. 산출물이 속한 프로젝트·서비스명을 사용한다. (예: `ServicePlex`, `Agent_Paw`)
. 시스템명 내부 공백은 언더스코어(`_`)로 치환한다.

### 2.3. 버전

. 형식 : `v[Major].[Minor].[Patch]` 또는 `v[Major].[Minor]`
. 버전 구분자는 반드시 점(`.`)을 사용한다. 언더스코어(`_`)를 사용할 수 없다.

| 버전 타입 | 변경 범위 | 예시 |
|---|---|---|
| Major | 전체 구조 개편, 파트 체계 재설계 | v1.0 → v2.0 |
| Minor | 섹션 신설, 모듈 추가, 대규모 보강 | v1.0 → v1.1 |
| Patch | 오탈자·토시 수정, 문구 정정 | v1.0.0 → v1.0.1 |

## 3. 파일명 예시

```
ServicePlex_시스템통합기획서_v0.6.2.md
ServicePlex_서비스정책서_v0.5.0.docx
ServicePlex_사용자_매뉴얼_v0.3.2.md
Agent_Paw_기획서_v1.0.md
Agent_Paw_API명세서_v2.0.md
```

## 4. 금지 사항

. 버전 구분자에 언더스코어(`_`) 사용을 금지한다. 반드시 점(`.`)을 사용한다.
. 파일명 내 공백 사용을 금지한다. 단어 구분은 언더스코어(`_`)를 사용한다.
. 특수문자(하이픈, 괄호 등) 사용을 금지한다.
. 버전 없이 파일명을 작성하는 것을 금지한다.
. 파일명에 날짜(YYMMDD/YYYYMMDD) 포함을 금지한다. 날짜는 메타데이터 또는 변경 이력에 기록한다.

## 5. 안티패턴

```
❌  ServicePlex_서비스정책서_v0_3_3.docx     ← 버전 구분자 언더스코어
❌  ServicePlex 서비스정책서 v0.3.3.docx     ← 공백
❌  ServicePlex_서비스정책서_260203.md       ← 날짜 포함 (구 IGS 양식)
❌  ServicePlex-서비스정책서-v0.3.3.md       ← 하이픈
✅  ServicePlex_서비스정책서_v0.3.3.docx
```

## 6. 구버전 보관

. 구버전은 `Docs/old/` 폴더로 이동하여 보존한다. 파일명은 변경하지 않는다. (CLAUDE.md §6.4 참조)
. 산출물 폴더(`Docs/`) 직속에는 각 산출물의 최신 버전 파일 1개만 노출한다.
