## 2. Appendix — 세부 업무 현황

큰 업무군(예: 이벤트 기획)은 건별 세부 리스트를 별도 매트릭스로 제공한다. 필요한 만큼 Appendix 섹션을 추가한다.

### Appendix A. [세부 업무군 명] — [N주차]

| 번호 | 프로젝트 / 건명 | 진행 업무 | 예정 업무 | 이슈 |
|---|---|---|---|---|
| 1 | [게임명(플랫폼)_이벤트명] | [단계 — 예: SB 작성 / QA 완료 / 오픈 완료(6/26)] | [단계 — 예: 킥오프 예정 / 업무 F/U / 오픈 예정(7/2)] | [이슈 내용 또는 `-`] |
| 2 | | | | |
| ... | | | | |
| 업무량 요약 | - | 총 N개 [단위], N건 [행위] 처리 | 총 N개 [단위], N건 [행위] 예정 | - |

표준 상태 표기 (진행 업무 / 예정 업무 셀에 사용):

. 요건 확인 → SB 작성 → 킥오프 예정 → 업무 F/U → QA 완료 → 배포 예정 → 오픈 완료(MM/DD)
. F/U = Follow Up. IGS 관례 그대로 사용한다.
. 건너뛴 단계가 있으면 비고에 사유를 명시한다.
