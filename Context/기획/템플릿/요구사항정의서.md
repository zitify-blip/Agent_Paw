| NO | INDEX | 구분 | 요구사항 | 비고 | 요건 협의사항 | 확인 필요 사항 | 참고문서 |

| 2 | [AD_ME_002] | 회원 | | | | | |

컬럼 사용 규칙:

. **NO** : 연속 번호. 추가 시 중간 삽입 금지, 말미 증가.
. **INDEX** : 기능 코드(예: `AD_ME_001`). 내부 추적용. 미정 시 `--` 로 표기.
. **구분** : §2 에서 정의한 값만 사용한다.
. **요구사항** : 명사구로 짧게. 상세는 옆 컬럼들에 분산.
. **비고** : 요구사항 셀을 보완하는 짧은 메모.
. **요건 협의사항** : 현업과 이미 합의된 정책·리스트·프로세스를 상세히 기술. bullet/번호 허용.
. **확인 필요 사항** : 개발·QA·외부 시스템 담당자에게 확인이 필요한 항목. 의문형 허용. 해소되면 협의사항으로 이동.
. **참고문서** : 파일명 + 버전 + 섹션 번호까지 명시.

## 4. 요건 상태 정의

| 상태 | 정의 |
|---|---|
| 신규 | 최초 등록 상태. 확인 필요 사항이 남아 있다. |
| 협의중 | 현업·개발 간 확인 필요 사항을 해소 중이다. |
| 확정 | 확인 필요 사항이 모두 해소되어 기획·개발 착수 가능하다. |
| 철회 | 요건이 취소되었다. 삭제하지 않고 철회 사유를 비고에 기술한다. |
