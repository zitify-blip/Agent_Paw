## 2. 인수 조건 (Acceptance Criteria)

### 시나리오 1: [정상 플로우]

**Given** (전제 조건)
- User가 로그인된 상태이다
- User의 일일 광고 시청 한도가 남아있다

**When** (행동)
- User가 "광고 시청" 버튼을 클릭한다

**Then** (결과)
- Client가 30초 광고를 재생한다
- 광고 시청 완료 후 Server가 [재화명]을 적립한다
- Client가 적립 완료 메시지를 출력한다
- Server가 일일 한도를 1 감소시킨다

---

### 시나리오 2: [예외 상황]

**Given** (전제 조건)
- User가 로그인된 상태이다
- User의 일일 광고 시청 한도가 0이다

**When** (행동)
- User가 "광고 시청" 버튼을 클릭한다

**Then** (결과)
- Client가 "오늘 광고 시청 횟수를 모두 사용했다" 메시지를 출력한다
- 광고가 재생되지 않는다
- Client가 다음 리셋 시간을 출력한다

---

### 체크리스트

- [ ] User가 광고를 시청할 수 있다
- [ ] 광고 시청 완료 시 Server가 [재화명]을 정확하게 적립한다
- [ ] Server가 일일 한도를 정확하게 차감한다
- [ ] 한도 초과 시 Client가 적절한 에러 메시지를 출력한다
- [ ] 광고 시청 중 앱을 종료해도 진행 상태가 유지된다
- [ ] 네트워크 오류 시 Client가 적절하게 처리한다

## 3. 페르소나

### 주 사용자: [페르소나명]

| 항목 | 내용 |
|------|------|
| **이름** | 김철수 (가명) |

**배경:**
- 출퇴근 시간에 지하철을 이용한다
- 자투리 시간을 활용해 용돈을 벌고자 한다

**페인 포인트:**
- 복잡한 절차를 선호하지 않는다
- 광고가 과도하게 긴 경우 이탈할 가능성이 높다

## 4. UI/UX 요구사항

### 화면 구성

#### 메인 화면
```
┌─────────────────────────┐
│  📱 광고 스테이션        │
├─────────────────────────┤
│                         │
│   [광고 시청하기]        │
│   ▶️ 30초 광고           │
│                         │
│   오늘 남은 횟수: 25/50  │
│                         │
└─────────────────────────┘
```

### 인터랙션

1. **버튼 클릭**
   - User가 "광고 시청하기" 버튼을 클릭한다
   - Client가 로딩 인디케이터를 출력한다
   - Client가 광고 화면으로 전환한다

2. **광고 시청**
   - Client가 전체 화면으로 광고를 재생한다
   - 스킵은 불가하다 (30초 필수 시청)
   - Client가 진행 바를 출력한다

3. **완료 처리**
   - Client가 적립 애니메이션을 출력한다
   - Client가 성공 메시지를 2초간 출력한다
   - Client가 메인 화면으로 복귀한다

### 디자인 가이드라인
- **버튼 색상**: Primary 색상 (#1e3a8a)
- **텍스트**: 14pt, Regular
- **아이콘**: 24x24px
- **간격**: 16px 표준 패딩

## 5. 기술 요구사항

### 프론트엔드
- **컴포넌트**: `AdStationScreen.tsx`
- **상태 관리**: React Context (AdContext)
- **API 호출**: `watchAd()`, `claimReward()`

### 백엔드
- **Cloud Function**: `recordAdView`
- **Database**: 
  - `users/{uid}/daily_limits` - 일일 한도 갱신
  - `wallets/{uid}` - [재화명] 잔액 갱신
  - `users/{uid}/transactions` - 거래 내역 생성

### 외부 연동
- **AdMob**: 보상형 동영상 광고 SDK
- **Analytics**: 광고 시청 이벤트 로깅

## 7. 테스트 시나리오

### 테스트 케이스 1: 정상 광고 시청
1. 로그인된 User로 광고 스테이션에 접속한다
2. User가 "광고 시청하기" 버튼을 클릭한다
3. User가 30초 광고를 전체 시청한다
4. [재화명] 적립을 확인한다
5. 일일 한도 감소를 확인한다

**예상 결과:** [재화명] [금액] 적립, 한도 1 감소

### 테스트 케이스 2: 한도 초과
1. 일일 한도를 모두 사용한 계정으로 로그인한다
2. User가 "광고 시청하기" 버튼을 클릭한다
3. 에러 메시지를 확인한다

**예상 결과:** "오늘 광고 시청 횟수를 모두 사용했다" 메시지 출력

### 테스트 케이스 3: 네트워크 오류
1. 광고 시청 중 네트워크 연결을 끊는다
2. 재시도 메커니즘을 확인한다

**예상 결과:** 재시도 안내 메시지 출력, 진행 상태 유지

## 8. 성공 지표 (KPI)

### 정량적 지표

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 일일 광고 시청 횟수 | 평균 30회/사용자 | Analytics |
