| 항목 | 내용 |
|------|------|
| **문서 제목** | 공통 섹션 — 메타데이터 |
| **문서 버전** | v1.1 |
| **작성일** | 2026-04-14 |
| **작성자** | 안승준 |
| **문서 상태** | ✅ 승인 완료 |
| **관련 문서** | 가이드/상태표기.md, 공통섹션/승인기록.md |

---

문서 상단에 테이블 형식으로 작성한다. 프로젝트명은 고정값이므로 메타데이터에 포함하지 않는다.

## 1. 필수 필드

. 문서 제목, 버전, 작성일, 작성자, 문서 상태

## 2. 표준 구조

```markdown
## 메타데이터

| 항목 | 내용 |
|------|------|
| **문서 제목** | [문서 제목] |
| **문서 버전** | v1.0 |
| **작성일** | YYYY-MM-DD |
| **작성자** | [작성자명] |
| **검토자** | [검토자명] |
| **승인자** | [승인자명] |
| **문서 상태** | 🔄 작성 중 / ✅ 검토 완료 / ✅ 승인 완료 |
| **관련 문서** | [관련 문서 링크] |
```

## 3. 문서 유형별 추가 필드

| 문서 유형 | 추가 필드 |
|---|---|
| API명세서 | API명, Base URL, 인증 방식 |
| 테스트케이스 | 테스트 대상, 테스트 유형, 우선순위, 테스트 환경 |
| 릴리즈노트 | 릴리즈명, 릴리즈 유형, 플랫폼 |
| 사용자스토리 | 스토리 ID, 에픽, 스토리 포인트, 스프린트 |
| 기획서 | 관련 부서, 우선순위, 예상 일정 |
| 기술명세서 | 모듈/기능명, 관련 기획서, 저장소 |
| 주간보고서 | 보고 기간, 소속, 보고 대상, 보고 유형 |
| 회의록 | 회의명, 회의 일시, 회의 장소, 회의 유형 |
| 제안서 | 제안 대상 시스템, 검토자 |
| 요구사항정의서 | 시스템명, 참조 문서 (파일명 + 버전) |
| 운영정책 | 적용 서비스, 시행일, 부칙 적용일 |
| 스토리보드 | 프로젝트명, 부서, 오픈 예정일 |
| 기능정의서 | 시스템명, 주요 사용자 |
| 협업프로세스 | 협업 대상 (부서 A ↔ 부서 B) |

## 4. 신규 IGS 계승 템플릿의 "상단 정보" 처리

다음 6종 템플릿은 IGS 시기의 양식을 계승하여 본문 첫 섹션에 자체 "상단 정보" 표를 둔다 — `제안서.md` · `요구사항정의서.md` · `운영정책.md` · `스토리보드.md` · `기능정의서.md` · `협업프로세스.md`. 또한 `주간보고서.md` · `회의록.md` 도 동일하게 자체 "상단 정보" 를 둔다.

. 이들 템플릿을 사용할 때는 본 문서의 "메타데이터" 표를 별도로 작성하지 않고, 템플릿 자체 "상단 정보" 표만 작성한다.
. "상단 정보" 표가 누락하는 메타데이터 필드(검토자·승인자·문서 상태·관련 문서)는 별도로 표 하단에 행을 추가하거나, `공통섹션/승인기록.md` 표로 이관한다.
. 이중 메타데이터 작성을 금지한다 (한 문서에 "상단 정보" 와 "메타데이터" 가 병존할 수 없다).
