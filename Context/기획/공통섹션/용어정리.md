문서 내 사용하는 전문 용어를 테이블로 정의한다. 프로젝트 공통 용어는 해당 문서에서 실제 사용하는 경우에만 포함한다.

## 1. 표준 구조

```markdown
## 용어 정리

| 용어 | 정의 |
|------|------|
| [용어] | [정의 — ~이다 어미로 종결] |
```

## 2. 작성 규칙

. 정의는 `~이다` 어미로 종결한다.
. 약어는 풀네임을 함께 표기한다. (예: TC | Test Case의 약어이다)
. 동일 용어를 여러 문서에서 사용할 경우 정의를 통일한다.

## 3. 부록 — 웹·게임 서비스 도메인 공통 약어

본 프로젝트 및 관련 산출물에서 반복 등장하는 약어를 정리한다. 문서 본문에서 해당 약어를 처음 사용할 때 풀네임을 병기한 후 약어만 사용한다.

| 약어 | 풀네임 | 정의 |
|---|---|---|
| GNB | Global Navigation Bar | 사이트 전역 상단 네비게이션 영역이다. |
| LNB | Local Navigation Bar | 특정 메뉴 하위에서 노출되는 좌측·지역 네비게이션 영역이다. |
| CI | Corporate Identity | 기업 통합 아이덴티티 이미지이다. |
| BI | Brand Identity | 브랜드·서비스 단위 아이덴티티 이미지이다. |
| SB | Storyboard | 화면 단위 기획 문서이다. |
| IA | Information Architecture | 정보 구조도이다. |
| FSD | Functional Specification Document | 기능 명세서이다. |
| PB | Program Behavior | 프로그램 동작 명세이다. |
| CRUD | Create / Read / Update / Delete | 데이터 생성·조회·수정·삭제 제약 명세이다. |
| CBT | Closed Beta Test | 비공개 베타 테스트이다. |
| OBT | Open Beta Test | 공개 베타 테스트이다. |
| F/U | Follow-Up | 후속 조치·진행 관리이다. |
| QA | Quality Assurance | 품질 검수이다. |
| PM | Project Manager | 프로젝트 관리자이다. |
| CS | Customer Service | 고객 지원이다. |
| GM | Game Master | 게임 운영자이다. |
| CM | Community Manager | 커뮤니티 운영자이다. |
| SVN | Subversion | 중앙집중형 소스 버전 관리 시스템이다. |
| RC | Release Candidate | 배포 후보 버전이다. |
